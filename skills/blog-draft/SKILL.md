# blog-draft スキル

## 前提確認

```bash
ls -la logs/retro-candidates.md 2>/dev/null || echo "retro-candidates.md なし"
```

`retro-candidates.md` がない場合は `/retro` を先に実行するよう案内して終了する。

## 実行手順

1. `logs/retro-candidates.md` を Read で読み込む

2. 候補を一覧表示してユーザーに選ばせる:

```
以下の候補があります。記事にするものを選んでください:

1. {候補1の一行サマリー}（推奨度: 高）
2. {候補2の一行サマリー}（推奨度: 中）
```

3. 選択された候補をもとに以下を決める:
   - **タイトル**: 技術記事らしい日本語タイトル
   - **slug**: タイトルを kebab-case 英語に変換（例: `claude-code-hook-tdd`）
   - **tags**: 内容に合う技術キーワード（3〜5個）

4. 以下のフォーマットで MDX を生成する:

````mdx
---
title: "{タイトル}"
date: {YYYY-MM-DD}
tags: [{tag1}, {tag2}]
draft: true
---

## はじめに

{解決した課題・背景を1〜2段落で。なぜこれを書くかを明確に。}

## 状況

{問題が起きた状況・環境・前提条件を簡潔に。}

## 詰まったポイント

{試行錯誤した内容・罠になった点を具体的に。コードブロックを積極的に使う。}

```言語
{コード例}
```

## 解決策

{鍵となった判断・手順を再現可能な形で。}

```言語
{コード例}
```

## まとめ

- {学び1}
- {学び2}
````

5. `drafts/{YYYY-MM-DD}-{slug}.mdx` に Write で保存する

## 品質チェック（保存前に確認）

- `draft: true` が設定されているか
- 個人情報・プロジェクト固有の情報（パス・社名・顧客名）が含まれていないか
- コードブロックに言語識別子が付いているか

## やってはいけないこと

- `draft: false` で保存すること
- retro-candidates.md の内容をそのままコピーすること（記事向けに書き直す）
- 1候補から複数記事を作ること
